## Local Call (Provider-mimic)

Use built-in `runfabric call-local` to start a local HTTP server that forwards provider-shaped requests to your handler.

```bash
npm run call:local
curl -i http://127.0.0.1:8787/hello
# stop server: Ctrl+C or type 'exit' and press Enter
npm run call:local -- --provider alibaba-fc --host 127.0.0.1 --port 8787 --serve --watch
npm run call:local -- --provider alibaba-fc --method GET --path /hello
npm run call:local -- --provider alibaba-fc --event ./event.json
```

Supported options for `call:local`:

- `--serve`
- `--watch`
- `--host <host>`
- `--port <number>`
- `--provider <id>`
- `--method <HTTP_METHOD>`
- `--path </route>`
- `--query key=value&key2=value2`
- `--header key:value` (repeatable)
- `--body <string>`
- `--event <path-to-json>`

- TypeScript mode: if no built handler is found, `call-local` runs an initial `tsc -p tsconfig.json` automatically (requires `typescript` in dev dependencies).

## Credentials

Set credentials for `alibaba-fc` in your shell before running deploy:

```bash
export ALICLOUD_ACCESS_KEY_ID="your-value"
export ALICLOUD_ACCESS_KEY_SECRET="your-value"
export ALICLOUD_REGION="your-value"
```

Or create `.env` in this project root:

```dotenv
ALICLOUD_ACCESS_KEY_ID=your-value
ALICLOUD_ACCESS_KEY_SECRET=your-value
ALICLOUD_REGION=your-value
```

Load `.env` and run:

```bash
set -a
source .env
set +a
npm run deploy
```
